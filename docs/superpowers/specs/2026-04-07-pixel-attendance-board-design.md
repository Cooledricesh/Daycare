# 픽셀 교실 출석 대시보드 (Pixel Classroom Attendance Board)

## 개요

레트로 픽셀아트 스타일의 시각적 출석 현황 대시보드. 호실별로 환자들의 출석 상태를 "교실" 메타포로 표현한다. 기존 기능에 영향 없는 독립 페이지.

## 목적

- 코디네이터가 한눈에 전체 호실 출석 현황을 파악
- 기존 리스트 뷰 대비 시각적 직관성 제공
- 재미 요소가 있는 부가 기능

## 라우트

- **경로**: `/staff/attendance-board`
- **네비게이션**: 스태프 사이드바에 "출석 보드" 메뉴 추가
- **접근 권한**: coordinator 역할

## 호실 카드 (RoomCard)

### 구조

```
┌─ 3101호실 ── 김코디 ── 8/12 출석 ─┐
│ ┌───┐ ┌───┐ ┌───┐ ┌───┐         │
│ │ 김 │ │ 이 │ │   │ │ 박 │         │
│ │ ○  │ │ ○  │ │ □  │ │ ○  │         │
│ └───┘ └───┘ └───┘ └───┘         │
│ ┌───┐ ┌───┐ ┌───┐ ┌───┐         │
│ │   │ │ 정 │ │ 최 │ │   │         │
│ │ □  │ │ ○  │ │ ★  │ │ □  │         │
│ └───┘ └───┘ └───┘ └───┘         │
└──────────────────────────────────┘
```

### 카드 헤더

- 호실 번호 (픽셀 폰트)
- 담당 코디네이터 이름
- 출석률 (출석 수 / 예정 수)

### 카드 배경

- 레트로 게임풍 연한 격자 패턴 (CSS `background-image` 반복 패턴)
- 호실별 미세하게 다른 배경색 (구분용)

### 좌석 그리드

- 카드 내부: 3~4열 그리드
- 예정된 환자만 표시 (비예정 환자는 표시하지 않음)

## 환자 좌석 셀 (SeatCell)

### 상태별 표현

| 상태 | 시각적 표현 | 색상 | 애니메이션 |
|------|-----------|------|-----------|
| 출석 완료 | 컬러 아바타 + 의자 | 환자 고유색 (이름 해시 기반) | 미니 바운스 (1회) |
| 예정/미출석 | 빈 의자 + 흐린 실루엣 | 회색 톤 | 느린 깜빡임 |
| 진찰 완료 | 출석 아바타 + 별 뱃지 | 초록 별 | 반짝임 |
| 과제 있음 | 아바타 + 느낌표 뱃지 | 주황 느낌표 | 없음 |

### 아바타 생성 규칙

- 환자 이름의 첫 글자를 이니셜로 사용
- 배경색: 환자 ID를 해시하여 미리 정의된 레트로 팔레트에서 선택
- 레트로 팔레트 (8색): `#e74c3c`, `#e67e22`, `#f1c40f`, `#2ecc71`, `#1abc9c`, `#3498db`, `#9b59b6`, `#e84393`

### 의자 디자인

- CSS로 구현하는 심플한 픽셀 의자
- `box-shadow` 또는 `border`로 도트 느낌 표현

## 인터랙션

### 환자 셀 클릭

- Popover로 상세 정보 표시:
  - 환자 이름 (전체)
  - 출석 시간 또는 "미출석"
  - 진찰 상태
  - 과제 상태

### 헤더

- 날짜 선택 (오늘 기본, 이전 날짜 조회 가능)
- 전체 출석률 프로그레스 바
- 수동 새로고침 버튼

### 자동 갱신

- React Query `refetchInterval: 30000` (30초)
- 해당 페이지에서만 활성화

## 픽셀아트 스타일

### 폰트

- 제목/숫자: `'DungGeunMo'` 웹폰트 (한글 지원 픽셀 폰트)
- 폴백: `'Press Start 2P'`, monospace

### CSS 스타일 가이드

```css

border-radius: 0; /* 둥근 모서리 제거 */

/* 픽셀 그림자 */

/* 이미지 렌더링 (픽셀 아트 선명하게) */
```

- 카드 배경: `#fff8e7` (밝은 크림)
- 테두리: `#5c4a3a` (갈색)
- 텍스트: `#2c2c2c` (진한 회색)
- 빈 좌석: `#d4d0c8` (레트로 회색)
- 출석 강조: `#4ade80` (초록)

## 데이터

### 사용하는 기존 API

1. **PatientSummary 목록** (GET `/api/staff/patients`)
   - `is_attended`, `attendance_time`, `is_scheduled`, `is_consulted`, `has_task`
   - 현재 room_number가 응답에 없으므로 추가 필요

2. **room_coordinator_mapping** (GET `/api/admin/room-mappings`)
   - 호실 → 코디네이터 매핑

### 필요한 API 변경

- `PatientSummary` 응답에 `room_number` 필드 추가
- 또는 별도의 board 전용 API 엔드포인트 신설 (추천)
  - GET `/api/staff/attendance-board?date=YYYY-MM-DD`
  - 응답: 호실별로 그룹핑된 환자 목록 + 출석 상태

### 프론트엔드 데이터 처리

- React Query로 데이터 페칭
- 프론트엔드에서 room_number 기준 그룹핑
- 코디네이터 정보는 room_coordinator_mapping에서 매칭

## 성능 고려사항

- Next.js 코드 스플리팅으로 해당 페이지 진입 시에만 번들 로드
- 웹폰트는 해당 페이지에서만 `@font-face` 로드
- 폴링은 페이지 포커스 시에만 활성 (`refetchOnWindowFocus`)
- 새로운 DB 쿼리 없음 (기존 API 재사용)

## 범위 외 (Out of Scope)

- 환자 개인 아바타 커스터마이징
- 실시간 WebSocket 푸시
- 출석 체크 기능 (기존 대시보드에서 수행)
- 관리자/의사 역할 접근 (코디네이터 전용)
